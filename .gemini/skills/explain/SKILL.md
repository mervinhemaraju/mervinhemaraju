# Explain: $ARGUMENTS

Give a layered explanation:

1. **One-sentence summary** — what it does at the highest level
2. **Context** — where it fits in the project/system
3. **How it works** — walk through the logic step by step
4. **Key design decision** — why it was built this way (if apparent from code or comments)
5. **Gotcha** — one thing that's easy to misunderstand or get wrong

Use ASCII diagrams for flows or structures where it helps.
Keep the explanation as short as it can be while still being complete.
